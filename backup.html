The Big Picture: Arsitektur Web Modern

1. Frontend (Client-Side)

Ini yang dilihat user di browser.

Teknologi: HTML, CSS, JavaScript (React, Vue, dll.)

Tugas: Nampilin data, nangkep input user, bikin interaksi.

2. Backend (Server-Side)

"Otak" di balik aplikasi.

Teknologi: PHP (Laravel), Python (Django), Node.js, dll.

Tugas: Logika bisnis, otentikasi, nyambung ke database.

3. Database

Tempat kita nyimpen data.

Teknologi: MySQL, PostgreSQL, MongoDB.

Tugas: Nyimpen dan ngambil data.

Gimana cara mereka "ngobrol"?

Melalui API (JSON)

Bagian 1: PHP

The Foundation

PHP itu bahasa yang "pemaaf" (forgiving), tapi ini bisa jadi jebakan.

Jadi developer PHP yang jago itu kuncinya disiplin.

PHP: The DO's (Yang Harus Dilakukan)

  • Pakai versi PHP terbaru (saat ini 8.x).

    Kenapa? Jauh lebih kenceng & aman.

  • Wajib pakai Composer untuk dependency management.

    Kenapa? Biar nggak `require 'library.php'` manual. Composer yang ngurusin semua library.

  • Selalu pakai PDO (PHP Data Objects) untuk koneksi database.

    Kenapa? Ini standar biar aman dari SQL Injection.

  • Ikuti standar koding PSR-12.

    Kenapa? Biar kode kalian seragam dan gampang dibaca tim.

  • Selalu validate dan sanitize input dari user.

    Kenapa? Prinsipnya: "Never trust user input."

PHP: The DON'Ts (Yang Harus Dihindari)

  • Jangan pakai fungsi `mysql_*` atau `mysqli_*` secara langsung.

    Kenapa? Ini udah jadul (deprecated) dan nggak aman.

  • Nyampur logika PHP & HTML dalam satu file (Spaghetti Code).

    Kenapa? Bikin pusing pas maintenance. Ini alasan kita pakai framework.

  • Matiin error reporting (`error_reporting(0)`).

    Kenapa? Gimana mau benerin bug kalau bug-nya nggak kelihatan?

  • Jangan "reinvent the wheel" (Membuat ulang roda).

    Kenapa? Nggak usah bikin fungsi login/routing sendiri. Ambil aja dari Composer.

Bagian 2: JavaScript

The Interaction

JavaScript sekarang bukan cuma buat "efek salju" di website.

Ini udah jadi bahasa pemrograman serius yang powerful banget.

JavaScript: The DO's

  • Pahami ES6+ (JavaScript modern).

    Contoh: `let`/`const` (lupain `var`), Arrow Functions (`=>`), `async/await`.

  • Pahami cara kerja Asynchronous.

    Kenapa? Hampir semua hal di web modern itu async (ambil data API, nungguin event).

  • Pakai Strict Mode (`'use strict';`).

    Kenapa? Ini 'maksa' kita nulis kodingan yang lebih rapi dan aman.

  • Pecah kodingan kalian ke dalam Modules (`import`/`export`).

    Kenapa? Biar file JS-nya nggak jadi satu file raksasa (gampang di-manage).

JavaScript: The DON'Ts

  • Kebanyakan manipulasi DOM secara langsung.

    Kenapa? `document.getElementById(...)` itu lambat & bikin berantakan. Makanya ada React/Vue.

  • Terlalu percaya client-side validation.

    Kenapa? Validasi JS itu buat UX (biar user nyaman). Validasi di backend (PHP) itu buat Security (biar aman).

  • Nyimpen data sensitif di client-side.

    Kenapa? Jangan pernah simpen API Keys di kodingan JS. Semua orang bisa lihat!

Bagian 3: Laravel

The Framework

"Laravel itu nggak gantiin PHP. Laravel itu cara paling keren & terstruktur buat pakai PHP."

Kenapa Sih Harus Laravel? (The "Magic")

Laravel itu pakai pola desain yang udah teruji:

MVC (Model-View-Controller)

Model: Ngobrol sama database (Eloquent).

View: Tampilan yang dilihat user (Blade templates).

Controller: "Polisi lalu lintas" yang nyambungin M dan V.

Hasilnya? Kodingan jadi rapi, nggak campur aduk.

Eloquent ORM

Ngomong sama database pakai bahasa manusia.

User::where('semester', 6)->get()

Artisan

Asisten pribadi kita di command line.

php artisan make:controller

Laravel: The DO's

  • BACA DOKUMENTASINYA.

    Serius. Ini best practice nomor satu. Dokumentasi Laravel itu salah satu yang terbaik.

  • Pakai Migrations buat ngatur skema database.

    Kenapa? Jangan ganti-ganti tabel lewat phpMyAdmin. Migrations itu ibarat 'Git' buat database.

  • Pakai Form Requests buat validasi.

    Kenapa? Biar logika validasi pindah dari Controller. Controller jadi bersih & fokus.

  • Pakai Eager Loading (`::with('relations')`).

    Kenapa? Biar nggak kena N+1 Query Problem. Ini biang kerok performa lambat.

Laravel: The DON'Ts

  • Nulis semua logika di Controller (Fat Controller).

    Kenapa? Controller itu harusnya 'kurus'. Logika bisnisnya pindahin ke Service Classes.

  • Nulis logika di file `routes/web.php`.

    Kenapa? File routes itu cuma buat daftar rute. Logikanya taruh di Controller.

  • Nge-query database di dalam loop (Blade/Controller).

    Kenapa? Ya ini namanya N+1 Problem (100 posts = 101 query).

  • Ngoprek file di dalam folder `vendor`.

    Kenapa? Perubahan kalian bakal hilang pas `composer update`.

Bagian 4: Beyond the Code

The Professional Mindset

Jadi developer itu bukan cuma soal ngoding.

Tools Wajib (Selain VS Code pastinya)

  1. Git (Version Control)

    "Kalau kodingan kalian nggak ada di Git, itu sama aja nggak ada."

    Ini alat kolaborasi tim, bukan cuma buat backup.

    Minimal: add, commit, push, pull.

  2. Terminal / Command Line

    Nggak usah takut sama layar hitam. Developer pro itu 'rumahnya' di terminal (Artisan, Composer, Git, NPM).

  3. Debugging Tools

    JS: `console.log()` itu oke, tapi Chrome DevTools (Debugger) itu jauh lebih powerful.

    PHP: `dd()` di Laravel udah enak, tapi Xdebug itu baru level pro.

"Clean Code" itu Best Practice Paling Penting

"Kodingan yang bagus itu kayak lelucon yang bagus. Nggak perlu dijelasin."
  • Kasih Nama yang Jelas:

    `$u` vs `$user`, `$data` vs `$activeUsers`. Pakai nama yang jelas.

  • DRY (Don't Repeat Yourself):

    Liat ada kodingan copy-paste? Stop. Bikin jadi fungsi.

  • SRP (Single Responsibility Principle):

    Satu fungsi = Satu tugas. Satu class = Satu tanggung jawab. Gitu aja.

Ingat: Kodingan kalian bakal lebih sering dibaca orang daripada ditulis.

Security itu Wajib, Bukan Pilihan

Mindset-nya: "Jangan pernah percaya input dari user."

SQL Injection (SQLi)

Solusi: Pakai PDO & Eloquent. (Jangan pakai raw query).

Cross-Site Scripting (XSS)

Solusi: Blade (`{{ $variabel }}`) otomatis aman.

Hati-hati kalau pakai `{!! $variabel !!}`.

Cross-Site Request Forgery (CSRF)

Solusi: Laravel udah nyiapin `@csrf` token.

Menuju Magang / Dunia Kerja

Apa sih yang dicari perusahaan dari mahasiswa?

  1. Problem Solving Mindset:

    Bukan "hapal syntax", tapi "gimana cara kalian mecahin masalah baru".

  2. Kemampuan Belajar Mandiri:

    Jago Googling itu skill. Kuat baca dokumentasi itu skill.

  3. Dasar yang Kuat & Kemampuan Kolaborasi:

    Paham OOP (ini dasarnya Laravel), Paham Git, Paham HTTP (GET, POST).

Kesimpulan

  1. Frameworks (kayak Laravel) bikin kita kerja lebih cepat, lebih aman, dan lebih terstruktur.
  2. Best practices (DRY, SRP) itu panduan biar kodingan kita gampang di-maintenance.
  3. Tools (Git, Composer) itu wajib hukumnya buat kolaborasi.
  4. Teknologi bakal terus ganti, tapi prinsip (Clean Code, Security, Problem Solving) bakal kepakai selamanya.
